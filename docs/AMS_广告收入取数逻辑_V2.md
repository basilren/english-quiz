# 项目报表——AMS广告收入取数逻辑（V2）

> **变更说明**：本版将 AMS 广告流水的项目段维护职责从**收入系统**迁移至**管报系统**，在管报系统内建立 AMS 招商流水与 TME 项目段的映射关系，管报报表取数逻辑相应调整。

---

## 一、需求背景

招商广告收入在 AMS 系统中入账时**不携带项目段信息**，需要依赖下游系统通过映射规则将收入归属到正确的 TME 项目段，以支撑项目级管报分析。

**上一版方案**：由收入系统维护 AMS 流水与项目段的映射关系，管报从收入系统取数。
**本版方案**：映射关系维护上移至管报系统，管报系统直接对接 AMS 招商流水，在系统内完成映射与报表取数。

---

## 二、核心变更点（与上一版差异）

| 维度 | 上一版方案 | 本版方案 |
|------|-----------|---------|
| 项目段维护系统 | 收入系统 | **管报系统** |
| 映射界面 | 收入系统 Mapping 界面 | **管报系统——项目段 Mapping 界面** |
| 管报取数来源 | 从收入系统取已带项目段的广告收入 | **从 AMS 流水直接取数 + 管报调整** |
| AMS 系统改造 | 需 AMS 配合输出项目段或改造流水 | **不涉及 AMS 改造**，主要是 TME 侧系统改造 |
| 数据一致性保障 | 收入系统内部校验 | **管报系统增加季度对比校验（流水 vs 账单）** |

## 四、详细功能需求

### 4.1 管报系统——项目段 Mapping 界面

**功能定位**：在管报系统内维护 AMS 招商流水项目描述与 TME 项目段的一一映射关系，作为后续报表取数的基础维度关联。

**输入字段**：
- **AMS 流水招商项目描述**：来自 AMS 流水的 `merchants_item` / `AMS项目描述` 字段，标识招商广告的签约项目或客户维度信息。
- **TME 项目段**：管报系统已有的项目段主数据，需与映射后的招商流水金额进行关联。

**映射规则**：
- 支持一对一映射（一个 AMS 项目描述 → 一个 TME 项目段）。
- 支持批量导入/导出映射关系（便于初始化及后续维护）。
- 支持历史映射版本追溯（如项目描述对应的项目段发生变更，需保留历史映射用于往期数据还原）。
- 新增 AMS 项目描述首次进入系统时，支持自动识别并提示财管人员进行映射维护。

**权限控制**：映射关系的增删改需由财管人员操作，支持审批流或操作日志记录。

---

### 4.2 项目报表——招商广告收入取数逻辑

#### 4.2.1 取数公式

```
招商项目广告收入 = AMS招商流水金额（剔税）+ 账面调整金额 + 管报调整金额
```

#### 4.2.2 各数据来源说明

| 数据项 | 来源 | 说明 |
|-------|------|------|
| **AMS招商流水金额（剔税）** | AMS 流水 | 筛选 `sale_type = 招商`，根据映射关系归集到对应 TME 项目段，剔除税金后计入 |
| **账面调整金额** | 管报系统识别 | 识别账面广告收入中来源为"手工调整"的部分，根据已有项目段直接取进项目报表 |
| **管报调整金额** | 管报调整模块 | 财管人员通过管报调整模块录入的调整数，需带项目段信息 |

> **关键原则**：管报系统对于账面中 AMS 来源的广告收入**不直接取账面数**，而是以 AMS 流水数据为准，通过映射归集项目段后取数。账面与流水的差异部分通过"账面调整金额"和"管报调整金额"进行 reconciling。

---

### 4.3 招商广告 AMS 分成成本计算

**逻辑链路**：
1. 通过广告位 Mapping 确定该 Plan ID / 广告位对应的**结算组**。
2. 根据结算组，从收入系统获取该结算组对应的**AMS 分成比例**。
3. 计算：`招商广告 AMS 分成成本 = 招商项目广告收入 × AMS 分成比例`。

> **注意**：分成比例数据仍存储在收入系统中，管报系统需通过接口或定期同步的方式获取结算组与分成比例的对应关系。

---

### 4.4 季度对比数据完整性校验

**校验逻辑**：
```
流水中招商相关 Plan ID 收入  vs  AMS 账单中对应 Plan ID 收入
```

**目的**：确保管报系统从 AMS 流水取到的招商收入数据与 AMS 账单侧的总金额在 Plan ID 维度上一致，及时发现漏取、重复取或映射错误。

**执行频率**：按季度执行，支持人工触发或系统自动跑批。

**差异处理**：
- 若流水 > 账单：检查是否存在重复映射或未入账的流水被取数。
- 若流水 < 账单：检查是否存在映射缺失或流水筛选条件遗漏。
- 差异结果输出给财管人员，作为数据质量检查依据。

## 五、业务流程（4 步）

### Step 1：管报系统取 AMS 招商流水
- 管报系统从 AMS 流水接口获取全量字段数据。
- 筛选条件：`sale_type = 招商`。
- 根据**项目段 Mapping 界面**维护的映射关系，将 AMS 项目描述映射到 TME 项目段。
- 剔除税金后，将招商流水金额归集到对应项目段。

### Step 2：管报系统识别账面手工调整部分
- 管报系统识别账面广告收入中来源标记为"手工调整"的数据。
- 这部分数据已携带项目段，直接取进项目报表。

### Step 3：财管通过管报调整模块调差异
- 财管人员对比 AMS 流水归集金额与账面实际入账金额，识别时间性差异或其他调整项。
- 通过管报调整模块录入调整金额，需指定项目段。
- **重点考虑**：如何便捷化手工调整？例如：调减本月某个项目收入时，可指定该笔调整在未来哪个期间调增（跨期间调整能力）。

### Step 4：AMS 分成成本计算
- 根据 Plan ID → 广告位 Mapping → 结算组 → 收入系统分成比例，计算招商广告的 AMS 分成成本。
- 分成成本同样需归集到对应项目段。

## 六、报表展示需求

### 6.1 默认展示
- 项目维度默认展示**累计全部期间**的数据（即从项目开始到当前查询时点的累计值）。

### 6.2 期间选择
- 支持用户选择具体期间（如月度、季度、年度）进行查看。
- 切换期间时，数据需根据所选期间动态刷新。

### 6.3 收入下钻（Drill-down）
- **必须明确区分数据来源**，尤其是流水部分：
  - 来源于 AMS 流水的收入
  - 来源于账面调整的收入
  - 来源于管报调整的收入
- 下钻到明细时，需展示每一笔数据的来源标识、Plan ID、映射的 AMS 项目描述、TME 项目段、金额、期间等关键字段。

---

## 七、手工调整便捷化需求（重点）

当前方案存在**流水数据与账面数据的时间性差异**，必须依赖财管人工调整。为降低人工操作成本，需支持以下便捷化能力：

1. **跨期间调整**
   - 支持调减本期某项目收入时，指定该笔金额在未来哪个期间调增。
   - 系统自动生成对应期间的调整分录，避免财管手动在多个期间重复录入。

2. **批量调整**
   - 支持按项目段、期间、调整原因等维度批量录入调整金额。

3. **调整模板**
   - 支持保存常用调整模板（如"月度时间性差异调整"），一键套用。

4. **差异提示**
   - 系统定期自动比对流水与账面差异，超过阈值时主动提示财管人员进行调整。

## 八、优缺点分析

| 维度 | 说明 |
|------|------|
| **优点** | 1. **不涉及 AMS 改造**：减少跨系统协调成本，主要是 TME 侧系统改造；<br>2. **管报系统闭环**：映射、取数、调整、校验都在管报系统内完成，降低系统间耦合；<br>3. **灵活性高**：管报系统可快速响应映射关系变化，无需等待收入系统排期。 |
| **缺点** | 1. **准确性挑战**：流水数据与账面数据存在时间性差异，必须依赖财管人工调整，调整量大时可能影响报表及时性；<br>2. **数据质量依赖映射维护**：若 Mapping 界面维护不及时或不准确，将直接影响项目报表准确性；<br>3. **系统职责边界变化**：收入系统不再维护项目段映射，需确保管报系统与收入系统间数据口径对齐。 |
